Question 4 – Input/Output Redirection Investigation

Input File
Command: echo "Linux Command Line Interface Assignment" > input.txt


A file named input.txt was created containing the text "Linux Command Line Interface Assignment".

Explanation:
The echo command created the input file and stored sample text that was later used for redirection.



Output Redirection
Command: cat input.txt > output.txt

The contents of input.txt were successfully redirected to output.txt using the (>) operator.

Explanation:
The > operator redirected the standard output into a new file. The contents of the input file were copied successfully.



Append Redirection
Command: echo "This line was appended." >> output.txt

The text "This line was appended." was added to output.txt using the (>>) operator without overwriting the existing content.

Explanation:
The >> operator appended new text to the existing file while preserving its previous contents.



Error Redirection
Command: ls nonexistentfile 2> error.txt

An invalid command (ls nonexistentfile) was executed, and the resulting error message was redirected to error.txt using the (2>) operator.

Explanation:
The 2> operator redirected the standard error generated by the invalid command into error.txt. As a result, the error message was stored in the file instead of being displayed on the terminal.




Observation

The (>) operator redirects standard output to a file and overwrites its contents if it already exists. The (>>) operator appends new data to an existing file. The (2>) operator redirects standard error messages to a separate file.



Conclusion

The experiment successfully demonstrated output redirection, append redirection, and error redirection using Unix/Linux commands.
